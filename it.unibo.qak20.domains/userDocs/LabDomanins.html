Introduction

We want to manage a tea-room that offers a number N (N=2) of teatable situated at fixed position in the central area of the room.
Each teatable can be in one of the following states:
  • clean: without food residue and sanitized
  • dirty: not clean
  • busy: occupied by a client;
  • free: not busy and not clean
  • available: free and clean
A waiter:
  1. assigns an available table to a client that makes a request to enter in the tea-room;
  2. deploys the client from the tearoom entrancedoor to the selected table.

Problem analysis (just to start ... )

The map of the room is already available (see tearoomexplored.txt).

Our application domain demands for a proper representation of knowledge about the world. To this end, we could exploit a declarative style of programming based on (tu)Prolog (for a general introduction, see for example AnIntroductionToProlog.pdf).

We note that during its work, also the waiter (represented by a ddr robot) could be in different 'logical' state, each related to a corresponding activity:
  • athome: waits for someting to do
  • movingto( cell(X,Y) ): move from it current position to cell(X,Y)
  • cleaning( table(N) ): cleans the teatable number N.

Initial state of the teatables

teatable( 1, available ).
teatable( 2, available ).
Other possible states: busy, free, dirty, clean

Initial state of the waiter

waiter( athome ).
Other possible states: serving( CLIENTID ), movingto( cell(X,Y) ), cleaning( table(N) )
